# 业务规则与计算逻辑

## 一、退款规则

### 1.1 退款金额计算

**计算公式**：
```
建议退款金额 = 实付金额 × (剩余次数 / 总次数)
```

**详细逻辑**：
1. 获取购买记录的总次数（`total_sessions`）
2. 计算已使用次数：`status=2`（已核销）的记录数
3. 计算已过期次数：`status=3`（已过期）的记录数
4. 计算剩余次数：`remaining = total - used - expired`
5. 计算建议退款金额：`suggested_amount = paid_amount × (remaining / total)`

**代码位置**：`TradeService.java` - `refundPreview()` 方法

**示例**：
- 购买月卡：¥300，30次
- 已使用：10次
- 已过期：5次
- 剩余：15次
- 建议退款：¥300 × (15/30) = ¥150

---

### 1.2 退款申请规则

**前置条件**：
1. 购买记录状态必须为"有效"（`status=1`）
2. 不能有处理中的退款单（防止重复申请）
3. 退款金额不能超过实付金额
4. 退款金额不能为负数

**退款流程**：
1. 顾客申请退款 → 创建退款记录（`status=1`）
2. 更新购买记录状态为"已退款"（`status=2`）
3. 更新剩余次数为"已退款"（`status=4`）
4. 店长/财务审批退款
5. 确认退款 → 退款金额返还
6. 拒绝退款 → 退款记录状态改为"已拒绝"

**代码位置**：`TradeService.java` - `applyRefund()` 方法

---

### 1.3 确认退款逻辑

**确认退款时**：
1. 如果是储值钱包支付 → 退款金额返还到钱包
2. 预收款冲销 → 清空预收款余额
3. 生成退款负收入记录（冲减收入）
4. 生成退款支出记录
5. 记录操作日志

**代码位置**：`TradeService.java` - `approveRefund()` 方法

---

### 1.4 退款率计算

**计算公式**：
```
退款率 = 退款金额 / 总销售额
```

**统计周期**：近30天

**代码位置**：`FinanceController.java` - `shop()` 方法

---

## 二、提成规则

### 2.1 提成类型

| 类型 | 说明 | 计算方式 |
|------|------|----------|
| 按次 | 每核销一次固定金额 | 提成 = 核销次数 × 固定金额 |
| 按流水比例 | 按收入的百分比 | 提成 = 收入 × 比例 |
| 固定金额 | 固定月薪 | 提成 = 固定金额 |

**字典**：`commission_rule_type`
- 1: 按次（PER_SESSION）
- 2: 按流水比例（REVENUE_PERCENT）
- 3: 固定金额（FIXED）

---

### 2.2 提成结算流程

1. 配置提成规则（`commission_rules` 表）
2. 财务触发结算 → 按员工汇总已完成的核销记录
3. 匹配规则计算提成
4. 生成提成结算记录（`commission_settlements`）
5. 确认发放 → 状态改为"已发放"

---

## 三、库存规则

### 3.1 库存预警

**预警条件**：
```sql
SELECT * FROM inventory i 
JOIN materials m ON i.material_id = m.id 
WHERE i.quantity <= m.min_stock
```

**说明**：当物料库存低于设定的最低库存时，触发预警。

---

### 3.2 出入库规则

**入库**：
- 增加库存数量
- 记录入库流水（`inventory_transactions`）

**出库**：
- 减少库存数量
- 记录出库流水
- 检查是否触发库存预警

**关联**：核销时自动扣减套餐BOM中的物料库存

## 六、优惠券规则

### 6.1 优惠券类型

| 类型 | 说明 | 计算方式 |
|------|------|----------|
| 固定金额 | 满减券 | 订单金额 - 固定金额 |
| 百分比 | 折扣券 | 订单金额 × (1 - 折扣比例) |
| 兑换券 | 免费兑换 | 订单金额 = 0 |

---

### 6.2 优惠券使用规则

**前置条件**：
1. 优惠券未过期
2. 订单金额满足最低消费要求
3. 优惠券库存充足
4. 顾客未超过使用限制

**使用流程**：
1. 顾客选择优惠券
2. 系统校验优惠券有效性
3. 计算优惠金额
4. 创建优惠券使用记录
5. 扣减优惠券库存

### 10.3 席位与店铺关系

- 一个商户可以购买多个席位
- 每个席位可以开设一个店铺
- 席位到期后，店铺功能受限
- 删除店铺时，释放席位
